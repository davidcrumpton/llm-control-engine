## Benchmarking

I ran the following command to benchmark the models:

```sh
❯ for model in $(llmctrlx model --list)
do
  # Pre-load without timing (or just ignore this output)
  llmctrlx bench -u 'warmup' -m $model > /dev/null

  # Now run the actual timed benchmarks
  llmctrlx bench -u 'Why is the sky blue?' -m $model
  llmctrlx bench -u 'Why is the sky blue?' -m $model
done
```

## Conclusion

The above script gives a good approximation of the latency of the LLM server. However, it is not a perfect solution since it does not account for the time it takes to load the model into memory. The benchmark results show that the first query to the LLM server is slower than the subsequent queries. This is because the first query triggers the LLM server to load the model into memory. The warmup period reduces the number of queries that are slower than the subsequent queries.
